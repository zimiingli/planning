range of tasks, from question answering and code generation to web
navigation and embodied decision making~\cite{yao2023react,
shinn2023reflexion, zhou2024lats}. Yet, many tasks remain beyond the reach of a single forward pass, requiring the model to evaluate alternatives, verify intermediate results, or search over candidate actions~\cite{yao2023tree, hao2023reasoning, snell2024scaling}. This has motivated \emph{test-time compute}, where additional
computation at inference time improves decision quality~\cite{snell2024scaling, wang2026flare,DBLP:journals/corr/abs-2506-12928}.

However, whether this assumption holds uniformly across heterogeneous agent environments has not been directly tested. Taking token entropy as the canonical signal adopted by most prior work, Figure~\ref{fig:intro_reversal} measures its correlation with optimizer utility across six agent environments: the common assumption is clearly violated on half of them, where high entropy instead indicates states on which extra compute is wasted. The signal--utility relationship therefore \emph{reverses direction} across environments, with a sharp consequence: \emph{the better an uncertainty-thresholding gate is calibrated, the worse it performs in mismatched settings}, because more accurate calibration translates to more precise harm-targeting (\S\ref{sec:ablation}). This raises a central question: \emph{if the signal--utility direction is not fixed, can adaptive compute still be principled?}

We trace this reversal to two coexisting state types with opposite signal--utility relationships. \emph{Information-poor} states (Type~I) arise when the agent lacks sufficient information; rollouts cannot synthesize information the environment itself does not provide, and high uncertainty predicts \emph{negative} value of computation. In contrast, \emph{decision-difficult} states (Type~D) arise when multiple viable options exist; additional computation helps explore alternatives, and high uncertainty predicts \emph{positive} value. Crucially, whether a given state is Type~I or Type~D depends on what the agent already encodes, so the mixture is determined jointly by the environment and the backbone---directly explaining why direction reverses along both axes. The same uncertainty signal carries opposite meanings depending on which type dominates, an instance of Simpson's paradox~\cite{simpson1951interpretation,kdd2023simpson}. This decomposition makes a falsifiable prediction---reversal is determined by the (environment, backbone) pair's mixture of Type~I and Type~D states---which we verify on held-out data (\S\ref{sec:theory-verification}). It also yields a sharper, information-theoretic implication: \emph{the uncertainty signal $\sigma$ is not a sufficient statistic for the value of computation}, so no policy of the form $g(\sigma(s))$---monotone or otherwise, calibrated or not---can deliver non-negative value across heterogeneous (environment, backbone) settings (Proposition~\ref{prop:necessity}). Fixed-direction gating is the strict-monotone special case of a structurally insufficient family.

We trace this reversal to two coexisting state types with opposite signal--utility relationships. \emph{Information-poor} states (Type~I) arise when the agent lacks sufficient information; rollouts cannot synthesize what the environment does not provide, so high uncertainty predicts negative value of computation. For example, a fact-verification agent that hesitates because retrieved evidence is ambiguous gains nothing from additional rollouts. \emph{Decision-difficult} states (Type~D) arise when multiple viable options exist; rollouts help explore alternatives, so high uncertainty predicts 
positive value. For example, a coding agent that hesitates because several valid implementations exist benefits from exploring them. 

Whether a given state is Type~I or Type~D depends on what the agent already encodes, so the mixture is determined jointly by the environment and the backbone. The same uncertainty signal carries opposite meanings depending on which type dominates, an instance of Simpson's paradox~\cite{simpson1951interpretation, kdd2023simpson}. This decomposition yields a formal implication: the uncertainty signal is not a sufficient statistic for the value of computation, so under the two-source model, no policy depending on uncertainty 
alone can deliver non-negative value across heterogeneous 
settings (Proposition~\ref{prop:necessity}). It also makes a falsifiable prediction, that reversal is determined by the mixture of Type~I 
and Type~D states, which we verify on held-out data 
(\S\ref{sec:theory-verification}). Recovering sufficiency requires augmenting the uncertainty signal with features that correlate with the latent state type, such as 
step count as a proxy for information accumulation.

As an existence proof that direction---not gate capacity---is the bottleneck, we instantiate the minimal direction-learning procedure consistent with this analysis: \textbf{DIAL} (\textbf{D}irection-\textbf{I}nformed \textbf{A}daptive \textbf{L}earning), a sparse linear gate fit on signal-agnostic exploration data. Across six environments and three LLM backbones, DIAL already outperforms every fixed-direction baseline at zero per-step inference overhead, and its full range of gating behavior---from aggressive triggering to near-total suppression---emerges from direction learning alone. The simplicity of the gate is the point: once the direction is learned, the simplest classifier suffices, indicating that the bottleneck for cross-setting adaptive compute lies in the assumption tested here rather than in the sophistication of the gate.
=======
As an existence proof that direction is a first-order bottleneck 
that simple gates can address once utility feedback is collected, we propose DIAL (\textbf{D}irection-\textbf{I}nformed \textbf{A}daptive 
\textbf{L}earning). DIAL collects unbiased state--utility pairs via randomized exploration, then fits a sparse linear gate over a small set of state features, optionally augmented by LLM-proposed task-specific signals, to learn the signal--utility 
direction for each (environment, backbone) setting. Across six environments and three LLM backbones, DIAL outperforms every fixed-direction baseline, with its gating behavior ranging from aggressive triggering to near-total suppression depending on the 
learned direction. Once the direction is correctly identified, 
even the simplest classifier suffices.
