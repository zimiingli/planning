Method: EAAG (Environment-Aware Adaptive Gating)

1

Explore

50 episodes, randomized gating (epsilon = 0.5)

Collect (state, utility) pairs

Unbiased signal estimation

→

2

Reason

LLM analyzes exploration data

Generates environment profile

Proposes task-specific features

→

3

Learn & Deploy

LASSO feature selection

Logistic regression gate

Training <1s, zero deploy cost

Feature Pool

Universal: step_count, token_entropy, state_length, action_entropy, num_available_actions, has_output

LLM-generated: price_mentioned, action_is_click (WebShop); evidence_count (HotpotQA)

LASSO selects from union of both pools

Why Intentionally Simple

Bottleneck = direction discovery, not gate complexity

MLP wrong direction: -37.0 pp (HotpotQA)

Logistic correct direction: Pareto-dominates all

direction >> signals >> gate complexity
