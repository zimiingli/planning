% ABSTRACT

\begin{abstract}
Large language model (LLM) agents benefit substantially from
test-time compute, yet applying expensive optimization to every
decision step is wasteful. Selective triggering methods address this by using uncertainty
signals to decide when extra compute is worthwhile. However, these
methods share an implicit assumption: a fixed mapping from signal
value to compute need. We show that this assumption is wrong. Across eight diverse
agent environments and three model backbones, we find that the
signal--utility direction \emph{reverses}: the same entropy value
that predicts rollout benefit in one environment predicts rollout
harm in another. We trace this to a conflation of two uncertainty
sources, \emph{information poverty} and \emph{decision difficulty},
and formalize it via a two-source model grounded in Simpson's
paradox. We prove that direction discovery is necessary for
cross-environment non-negative value of computation, and propose EAAG
(Environment-Aware Adaptive Gating), which discovers
environment-specific gating patterns through exploration, LLM-based
reasoning, and sparse direction learning. EAAG outperforms all
fixed-direction baselines across eight environments while adding zero
per-step cost, with adaptive gating behavior emerging from direction
learning alone.
\end{abstract}
